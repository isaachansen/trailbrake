# Full app audit — 2026-07-01

Scope: Rust backend (`src-tauri` + all 5 crates), frontend store/data layer, all 29 widgets
(code + rendered screenshots at default/min/light-bg), manager UI, build & config health.
Ten independent audit passes, findings deduped and verified against source. No fixes applied.

**Build health: fully green.** `tsc --noEmit`, `cargo check --workspace` (zero warnings),
and `npm run build` all pass. Versions in lockstep at 0.2.2 across package.json /
tauri.conf.json / Cargo.toml / Cargo.lock. Updater endpoint verified correct; the weekly
trackmaps release can't hijack `/latest` (`makeLatest: false`, `prerelease: true`).

Findings marked **[×2]** were found independently by two auditors.

## 1. Critical / major — broken behavior

### Backend (Rust)

- **B1. Wrong session type all weekend.** `crates/iracing-connector/src/session.rs:337-366` +
  `connector.rs:806` — `scan_session_type` returns the *first* session under
  `SessionInfo.Sessions[]` and the connector never reads live `SessionNum`. In a
  Practice→Qualy→Race weekend, `session_type` says "Practice" the whole event, so every
  session-type-gated widget misbehaves in Qualifying and the Race.
- **B2. Shared-memory bounds never validated → app-killing crash possible.**
  `irsdk/header.rs:57`, `irsdk/mmap.rs:129`, `connector.rs:570-599` — offsets/lengths from
  the sim-written header are used unchecked (`buf_offset as usize` with no `.max(0)`; a
  negative i32 becomes ~1.8e19). A torn header during sim startup/crash, or any process
  squatting the unsecured `Local\IRSDKMemMapFileName` section, causes an access violation
  that takes down the whole app.
- **B3. Per-session state never reset on session change.** `connector.rs:568-594` —
  `fuel_history`, `prev_lap`, `prev_fuel`, `prev_flags`, `messages` survive a car/track
  change without app restart: fuel-per-lap skewed by a phantom crossing for up to 20 laps;
  Race Control shows last session's flags.
- **B4. Duplicate frames at ~200 Hz when the data-valid event can't open.**
  `connector.rs:549-556` — the claimed tick-count polling fallback doesn't exist; `poll()`
  sleeps 5 ms and unconditionally emits, flooding IPC/React with identical frames.
- **B5. Unbounded reader→bridge channel.** `crates/overlay-core/src/reader.rs:59` —
  60 Hz producer, consumer does JSON+IPC per frame; a webview stall grows memory without
  bound then replays stale frames instead of dropping to latest.

### Store / data layer (TS)

- **S1. Settings panel shows non-effective values.** `src/components/SettingsPanel.tsx:163-217`
  binds to raw `instance.opacity/scale/showIn`, but fresh widgets inherit globals
  (`layout.ts:124-143` sets `useGeneral* = true`). The panel misrepresents applied state and
  jumps on first interaction.
- **S2. Idle-preview mock persists real state.** `OverlayApp.tsx:46-59` feeds the mock's
  `carName: "Ferrari 296 GT3"` (a real car name, `mockSource.ts:257`) into
  `layoutStore.handleCar` → per-car profile auto-switch + persist to disk. Fake data
  mutating persisted state. (Related: mock roster says the player drives a BMW M4 GT3 EVO —
  `mockSource.ts:77` — so mock data is also self-inconsistent.)
- **S3. Stale/fake telemetry never cleared.** No path nulls `slow`/`caps`/`latestFast`
  (`store.ts`); toggling demo-data off leaves the last mock frame rendered forever.
- **S4. Cross-window sync is whole-blob last-writer-wins.** `layout.ts:201-206, 266-285` —
  any edit in one window within the other's 120 ms debounce silently overwrites it, and the
  pending save persists the loss. No merge/versioning; also no state-request handshake on
  overlay startup (broadcasts before listeners register are dropped), and `init()` can
  clobber fresher synced state with stale disk contents (`layout.ts:320-345`).
- **S5. [×2] Edit hotkey: unregister-before-register + persist-before-validate + swallowed error.**
  Rust `main.rs:588-603` unregisters the old shortcut first; if the new one is taken, no
  hotkey exists at all. Frontend `appSettings.ts:163-168` persists the accelerator first and
  `.catch(() => {})`s the failure — broken hotkey persists across launches with zero feedback.
- **S6. [×2] `overlay_visible` status omits the VR term.** `main.rs:503` computes
  `… || vr` but `emit_status`/`get_status` (535/648) drop it. With VR on and no session, the
  manager says hidden while the window is shown — and the idle-mock gate never fires, so VR
  panels show empty widgets even with demo data enabled.

### Widget logic

- **W1. [×2] Lap Timer PRED math wrong.** `LapTimer.tsx:58` — `predicted = cur + delta`
  uses the *running lap clock*; PRED ramps from ~0 to the lap time every lap. Should be
  `bestLapS + delta` (and null with no best).
- **W2. Delta shown with no best lap (honesty violation, two widgets).**
  `mockSource.ts:250` emits `deltaBestS` while `bestLapS` is null → Lap Timer shows
  DELTA +0.33 next to BEST "--", Delta Bar shows "+0.33 vs BEST". Fix mock *and* gate
  defensively in `LapTimer.tsx:51` / `DeltaBar.tsx:28-32` (a real sim could do the same).
- **W3. Slow Car Ahead fabricates distance.** `SlowCarAhead.tsx:56,123` — time gap × hardcoded
  42 m/s shown as precise "{n}m" and compared against a meters threshold. 20%+ off at slow
  tracks; violates never-fake-data. Show seconds or hide.
- **W4. Standings fabricates gaps for no-data cars.** `Standings.tsx:274-281` — null
  `gapToPlayerS` coerced to 0: a disconnected car gets a concrete GAP and poisons the INT
  below it. Also renders literal "#null" for numberless cars in mixed rosters (line 180).
- **W5. Heart Rate live path unreachable from UI.** `HeartRate.tsx:116` — `sessionId` is an
  enum with a single `""` option; both settings surfaces render enums as fixed selects, so
  HypeRate can only be enabled by hand-editing saved JSON.
- **W6. [×2] Tachometer shift point ignores car profile.** `Tachometer.tsx:51-55` — redline
  from profile, `shiftRpm` always config-default 8000; cars with lower redlines never show
  the amber shift band (green→red jump). Also the catalog description promises "analog rev
  counter with redline and shift-point markers"; it renders a bar with no markers
  (`widgetMeta.ts:21`).
- **W7. [×2] Garage Cover copy lies.** `GarageCover.tsx:65,68` — headline hardcoded
  "GRID FORMING" regardless of session; "SESSION STARTS IN" shows the current session's
  time *remaining*, not a start countdown.
- **W8. Corner Name shows the corner behind you.** `CornerName.tsx:103-135` — labels the
  last-*passed* turn with progress toward the next; billed as "the corner you're
  approaching". (Draft widget — fix before shipping.)

### Manager UI

- **M1. Widget preview lies about opacity.** `WidgetPreview.tsx:66-78` (and
  `WidgetGallery.tsx:101-115`) apply opacity as CSS element opacity over a fixed 0.55-alpha
  surface; the real host (`WidgetHost.tsx:39-40,114-121`) applies it as panel-background
  alpha keeping text crisp. The opacity slider is tuned against a preview that's wrong at
  every value.
- **M2. Hex color input is effectively read-only.** `ColorWheel.tsx:151-163` — controlled
  value + commit-only-on-full-match regex; intermediate keystrokes snap back. Only a
  select-all paste works. Affects accent picker and all per-widget color fields.
- **M3. "Reset options" doesn't restore size.** `WidgetConfigEditor.tsx:65-77` resizes on
  section toggles, but `layoutStore.resetConfig` (`layout.ts:400-410`) swaps config back
  without the reverse delta or `clampSize` — restored content renders squeezed.
- **M4. VR enable failure is silent.** `SettingsPage.tsx:287` discards the error string
  `setVrEnabled` returns; toggle snaps back with no explanation, status may still say "Ready".
- **M5. "Create profile" replaces your layout with the stock one, silently.**
  `ProfilesPage.tsx:104-116` + `layout.ts:425-436` — new profile = `defaultLayout()`,
  immediately activated; duplicate names are silently no-ops (rename too).

## 2. Systemic root causes (one fix, many symptoms)

- **T1. Glass surface too transparent on light footage.** `theme.ts:70`
  `surface: rgba(18,20,27,0.55)` — over bright scenes the panel blends to mid-gray and
  `textDim`/`textDim2`/`cell`/`gridLine` tokens collapse. Reported independently by 4 of 5
  visual reviewers across ~15 widgets (lap-timer's main clock, sector-delta near-total
  washout, weather, racecontrol, chat, tachometer track, etc.). Fix once: raise surface
  alpha to ~0.75–0.8 or add `brightness(0.55)` to `panelBlur`.
- **T2. White-alpha strokes on canvas widgets vanish on light bg.** TrackMap outline
  (`TrackMap.tsx:159-160`), Flatmap lap line/ticks (`Flatmap.tsx:93,101`), Traffic bar track
  (`TrafficIndicator.tsx:80`). Radar already solved this with mid-grey strokes
  (`Radar.tsx:141`) — adopt that pattern (or a token).
- **T3. `WidgetTitle` right slot can't shrink.** `WidgetTitle.tsx:26` — no `minWidth: 0` on
  the right wrapper, no `nowrap` on the title span: long right-slot text makes the *title*
  wrap and steal content height (TrackMap min-size breaks visibly). One fix helps every
  right-slot widget.
- **T4. Padding/header drift off the family tokens.** Hardcoded outer padding instead of
  `theme.widgetPad` in at least: Standings ("4px 5px"), Relative ("6px 7px 7px"),
  SectorDelta, DeltaBar, PitBoard, FuelSession, LaunchAssist, SetupComparison, RaceControl,
  Chat, Radar/TrackMap/Flatmap (three different hand-rolled values). Hand-rolled headers
  instead of `WidgetTitle`: PitBoard, LaunchAssist, Chat (unique purple dot).
- **T5. Mock data quality.** Mock drives screenshots, previews, and (per S2) even persisted
  state: player car name mismatch (BMW roster vs "Ferrari 296 GT3" carName), delta without
  best (W2), `class_position` computed against the wrong class + `total_cars: Some(1)`
  (`crates/overlay-core/src/mock.rs:431,491`), InputGraph history not pre-seeded so the
  gallery shot is 80% empty.

## 3. Minor bugs — backend

- `total_cars` includes the pace car while `cars` filters it (`connector.rs:809` vs 760) —
  off-by-one field count.
- VR double-start TOCTOU in `vr_set_enabled` (`vr.rs:68-86`) — three separate lock
  acquisitions around a seconds-long SteamVR init.
- `reconcile_overlay` read-then-apply race (`main.rs:495-517`) — interleaved hide/show can
  leave overlay invisible but cursor-captured with edit=true.

  blocker for a strict CSP). Overlay window capability grants updater/process/global-shortcut

  later orphans app-data dirs/update continuity).
